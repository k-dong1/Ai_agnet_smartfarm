# Tomato SmartFarm Pest Risk AI Agent

토마토 스마트팜 환경 데이터와 병해충 기준 정보를 연동하여, PyTorch 기반 MLP(인공신경망) 모델이 현재 온실 내 환경의 병충해(잿빛곰팡이병, 담배가루이) 위험도를 분류하고, AI Agent가 원인 분석과 대응 지침을 수립하며 로컬 LLM(Ollama)이 자연어 요약 설명을 제공하는 **스마트팜 병해충 위험 진단 AI 에이전트 시스템**입니다.

이 프로젝트는 프로토타입이며, 복잡한 웹 프레임워크나 가상 데이터를 배제하고 **실제 공공데이터 API 및 공공 사양서 규격**을 사용하여 정적 HTML 대시보드로 리포트를 자동 발행합니다.

---

## 1. 프로젝트 개요
스마트팜 내부 센서 환경값(온도, 습도, CO2, 광량, 일교차, 고습 누적 지속 시간 등)에 따라 토마토의 주요 병해충인 **잿빛곰팡이병**과 **담배가루이**의 감염 및 증식 위험률이 크게 좌우됩니다.
본 에이전트는 기 수집된 실제 스마트팜과 인근 농업기상관측 데이터를 연동하여 MLP 신경망 모델을 지도학습시키고, 최신 환경 정보가 유입될 때 실시간 위험 등급 분류(정상, 주의, 경고, 심각) 및 상세 원인 파악, 그리고 최적의 방제 가이드를 생성합니다.

---

## 2. 사용한 공공데이터 목록
실제 가동을 위해 다음 3가지 공공데이터 API와 직접 연동하고 데이터를 정제하여 사용합니다:
1. **스마트팜 데이터 마트 품목별 데이터셋 제공 서비스** (농림축산식품부)
   - 스마트팜 온실 내 센서 수집 정보 (온도, 습도, 이산화탄소, 광량)
2. **작물 병해충 검색 서비스** (농촌진흥청 NCPMS)
   - 토마토 대상 병해충(잿빛곰팡이병, 담배가루이)의 발병/증식 기준 조건 정보 검색
3. **농업기상 기본(V3) 및 상세(V4) 관측데이터 조회** (국립농업과학원)
   - 인근 농업기상관측소의 온도, 습도, 결로 시간, 풍향/풍속 시계열 데이터 수집

## 6. Ollama 실행 방법
이 시스템은 분석 결과를 사용자 친화적으로 요약하기 위해 Ollama 로컬 LLM을 보조 도구로 활용합니다.
1. [Ollama 공식 홈페이지](https://ollama.com/)에서 Ollama를 설치합니다.
2. 터미널을 열고 필요한 모델을 내려받은 뒤 가동합니다:
   ```bash
   ollama pull qwen2.5:3b
   ```
3. Ollama 서버가 로컬포트 `11434`에서 백그라운드로 작동하고 있는지 확인합니다.
*(참고: Ollama 연결에 실패하거나 모델이 설치되어 있지 않더라도, 전체 시스템은 중단 없이 정상 작동하며 안전한 Fallback 설명으로 대체됩니다.)*

## 8. MLP 구조
인공신경망은 온습도 및 변화량에 따른 비선형 위험 패턴을 정교하게 판별하기 위해 3개 레이어로 설계되었습니다.
- **Input Layer**: 7차원 입력 (온도, 습도, 광량, CO2, 고습 지속 횟수, 온도 변화량, 습도 변화량)
- **Hidden Layer 1**: 16 노드 (활성화 함수: ReLU)
- **Hidden Layer 2**: 8 노드 (활성화 함수: ReLU)
- **Output Layer**: 4 클래스 분류 (정상, 주의, 경고, 심각)

---

## 9. Adam/ReLU 선택 이유
- **ReLU (Rectified Linear Unit)**: MLP의 은닉층 활성화 함수로 ReLU를 사용했습니다. ReLU는 양수 입력은 그대로 통과시키고 음수 입력은 0으로 처리하여 비선형성을 부여하며, 역전파 시 기울기 소실(Vanishing Gradient) 문제를 방지하고 센서 기반 분류 문제에서 학습이 비교적 빠르고 안정적입니다.
- **Adam (Adaptive Moment Estimation)**: Optimizer는 Adam을 사용했습니다. Adam은 경사하강법 진행 시 모멘텀(Momentum)과 RMSProp의 장점을 결합하여 학습률(Learning Rate)을 파라미터별로 적응적으로 조절하므로, MLP 학습 시 손실 함수 공간에서 빠르고 안정적인 수렴을 돕기 때문에 선택했습니다.

---

## 10. 라벨링 기준
공공데이터에는 병해충 위험도 정답 라벨이 없으므로, 본 프로젝트에서는 잿빛곰팡이병 발생 조건과 스마트팜 환경 기준을 바탕으로 규칙 기반 라벨을 생성하여 MLP 학습에 사용했습니다.

위험도 판단 점수(Score) 합산 기준:
- 습도 >= 70.0% (주의 임계치): +1 점
- 습도 >= 90.0% (심각 임계치): +1 점
- 온도 <= 15.0℃ (저온 임계치): +1 점
- 고습 지속 횟수 >= 3회: +1 점
- 습도 변화량 >= 10.0%: +1 점
- 온도 변화량 >= 5.0℃: +1 점

최종 위험 등급 판정:
- **정상 (0)**: 누적 점수 1점 이하
- **주의 (1)**: 누적 점수 2점
- **경고 (2)**: 누적 점수 3 ~ 4점
- **심각 (3)**: 누적 점수 5점 이상

## 11. AI Agent 역할
스마트팜 AI 에이전트는 기계적인 신경망 분류 모델의 한계를 극복하고 사용자와 적극 소통하기 위해 설계되었습니다.
1. **실시간 위협 진단**: 유입된 최신 센서 피처를 MLP 모델에 제공하여 Softmax 예측 확률과 위험도 판정을 수행합니다.
2. **원인 추론 (Diagnostics)**: 입력 피처와 누적 시계열 추이를 역추적하여 "저온다습 조건 유입", "습도 급변" 등 구체적인 원인 리스트를 규명합니다.
3. **행동 추천 (Recommendations)**: 판정 등급과 원인에 발맞추어 환기 장치 기동, 야간 난방 최적화, 긴급 약제 방제 가이드라인을 동적으로 추출합니다.
4. **자연어 설명 번역**: 수치 결과와 전문 지침들을 로컬 LLM을 경유하여 비전문가인 농장 관리자도 즉각 직관적으로 이해할 수 있는 자연어 브리핑 문장으로 번역합니다.

## 12. HTML 리포트 확인 방법
`main.py` 실행 완료 후 생성된 `output/report.html` 파일을 웹 브라우저(Chrome, Edge 등)로 직접 열어 확인합니다.
- 다크 테마 기반의 고급스러운 카드 디자인(Glassmorphic Card Layout)이 내장되어 있습니다.
- 예측 위험 등급(정상, 주의, 경고, 심각)에 맞춰 대시보드의 네온 글로우 및 강조 색상이 동적으로 변합니다.
- 등급별 Softmax 확률이 비주얼 프로그레스 바 형태로 렌더링되며, 실시간 에이전트 단계별 실행 로그와 모델 학습 지표를 한눈에 모니터링할 수 있습니다.

## 13. 한계점
본 프로젝트는 실제 병충해 진단 시스템이 아니라 공공데이터 기반 위험도 판단 프로토타입입니다. 공공데이터에 병충해 위험도 정답 라벨이 직접 포함되어 있지 않기 때문에, 병해충 발생 조건을 참고한 규칙 기반 라벨링을 적용해 지도학습용 데이터셋을 구성했습니다. 따라서 실제 농가 적용 시에는 추가적인 도메인 전문가 검증이 필요합니다.
